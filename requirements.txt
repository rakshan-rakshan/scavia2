# Brigade Gateway Voice Lead Qualifier — Python dependencies

# pipecat is pinned to an EXACT version on purpose. Two facts force this:
#   1. pipecat moves module locations between minor releases. Pin 0.0.108
#      exposes `pipecat.services.sarvam.stt` (the STT service); earlier pins
#      such as 0.0.92 ship only `pipecat.services.sarvam.tts` and would break
#      `app/transports.py:create_stt_service`.
#   2. pipecat's [sarvam] extra pins `sarvamai` exactly (0.1.26 for 0.0.108).
#      Adding our own `sarvamai>=…` line makes pip backtrack pipecat to an
#      older, STT-less release. So we DO NOT override sarvamai — we let the
#      [sarvam] extra pull the version pipecat was tested against.
#
# On saaras:v3 (was pipecat issue #3783): the SarvamSTTService in 0.0.108
# already accepts `mode` and `prompt`, and pins sarvamai 0.1.26, so saaras:v3
# is supported out of the box. (There is no published sarvamai 0.1.30 — the
# previous >=0.1.30 pin was unsatisfiable and blocked every install.)
#
# Verified 2026-06-23 in a clean venv: install resolves, `pip check` is clean,
# all import paths resolve, and the FastAPI app boots (/health 200).
# After `pip install -r requirements.txt` you can re-check with:
#   python -c "import pipecat, sarvamai; print(pipecat.__version__, sarvamai.__version__)"
#   # expect: 0.0.108 0.1.26
